# Super RLL 概要

> Ralph-Lisa Loop — CLIゲートでAI生成コードの信頼性を保証する2エージェントコラボレーションツール。

## 目的

Super RLLが解決する根本的な問題：**AIが書いたコードを、誰がチェックするのか？**

単一のエージェントがコードを書き、かつ「完了」と判断するのは、自分の答案を自分で採点するようなものです。RLLは開発とレビューを2つの独立したエージェント——Ralph（開発者）とLisa（レビュアー）——に分離し、CLIゲートを審判として交互に作業します。人間はアーキテクチャの意思決定権を保持します。

すべての行やコミットをレビューする必要はありません。レビューは自動化され、人間の注意は重要な判断にのみ向けられます。

## 能力

- **ターン制開発ループ**：Ralphが作業を提出 → Lisaが独立してレビュー → 合格または修正 → 合意まで反復
- **自動品質ゲート**：テスト結果、ドキュメント同期、複雑度評価、安全監査——提出時に自動検証
- **マルチタイプfast-path**：コード、ドキュメント、レビュー、プロセスタスクに最適化されたワークフロー
- **自律モード**：エンジンモードが両エージェントを自動駆動、tmux不要
- **テストティアカスケード**：unit → smoke → integration → e2e、失敗時に自動ループバック

## 実行モデル

```
あなた（人間）→ 方向性の決定、アーキテクチャ判断
  ↓
Ralph（開発エージェント）→ 計画、コード、テストを作成
  ↓
CLIゲート → テスト結果、ドキュメント整合性、安全性を自動検証
  ↓
Lisa（レビューエージェント）→ 独立レビュー、テストの真正性を確認、PASSまたはNEEDS_WORKを返す
  ↓
Ralphに戻る → 修正または合意
```

全ラウンドが`.dual-agent/history.md`に記録され、完全なトレーサビリティを確保します。

## RLLとは何か、何でないか

**RLLは**：構造化されたAI支援開発ツールです。「AIがコードを書く」ワークフローを検証可能・追跡可能・信頼できるものにします。

**RLLではないもの**：完全自律型の開発ロボットではありません。人間の仲裁者は必須です——アーキテクチャの判断、セキュリティ境界、ビジネス方針はあなたが決めるものです。

公開ドキュメントは保守的です。RLLの能力記述は実際にshipされたCLI機能とゲートメカニズムに基づき、未リリースの機能を約束しません。

## ソースエビデンス

すべての能力記述は`cli/src/`下の実際の実装に対応しています：
- ターンプロトコル：`cli/src/commands.ts` — `cmdSubmitRalph` / `cmdSubmitLisa`
- 品質ゲート：`cli/src/policy.ts` — `checkRalph` / `runGate`
- テストカスケード：`cli/src/commands.ts` — `runTierCascade`
- 複雑度判定：`cli/src/complexity-judge.ts`
